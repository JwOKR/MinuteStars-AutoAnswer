## 油猴脚本 v4.3.0 迭代优化

基于真实页面 HTML（`考试@彭辉_2022年9月26日.txt`，30题，单选/多选/判断三类）对脚本进行针对性优化：

### 关键页面结构发现
- 题目容器：`div.answer` → `div.title`（题干）+ `div.radio`/`div.checkbox`（选项列表）
- 选项 DOM：`<div class="radio"><label><input type="radio" name="questionId" value="A"> A. 选项文字</label></div>`
  - input 在 label 内部，文字是 TextNode（`nextElementSibling` 为 null）
- 提交按钮：`#btnSubmitPapers`（交卷）、`#btnSavePapers`（保存答题）
- 判断题格式：与单选题相同，只是选项为 `A.对` / `B.错`

### 修改内容（v4.3.0）

1. **提交按钮修复**：`doSubmit()` 优先使用 `#btnSubmitPapers`；新增 `doSave()` 函数调用 `#btnSavePapers`
2. **getQText 优化**：清理出题人括号注释（如 `（王淋）`、`（李茂华）`），提升模糊匹配命中率
3. **fill() 重写**：新增 `getOptionText(inp)` 通过 `inp.closest('label')` 提取选项文本（适配 input 在 label 内的结构）；判断题支持兜底策略（A=对/B=错）
4. **ruleInfer 增强**：优化判断题识别正则（匹配 `A. 对` 等格式）；新增"以上都不是"反向逻辑
5. **collectAnswers 增强**：新增 checked/defaultChecked 退化策略；规范答案格式；采集后调用 refreshStats()

- **输出文件**：`千寻宜MinuteStars自动答题_merged.user.js` v4.3.0

## v4.3.1 Bugfix（17:01）
- 错误：`TypeError: Failed to construct 'MouseEvent': Failed to read the 'view' property from 'UIEventInit': Failed to convert value to 'Window'`
- 根因：Tampermonkey 沙盒 `window` ≠ 页面元素的 `ownerDocument.defaultView`，构造 MouseEvent 时传入沙盒 window 导致类型不匹配
- 修复：`checkInput` / `uncheckInput` 中改为 `view: input.ownerDocument.defaultView`
- 版本升至 v4.3.1
