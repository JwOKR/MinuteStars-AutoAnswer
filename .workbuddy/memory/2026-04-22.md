## 油猴脚本 v4.2.0 Word文档一键导入

在题库管理弹窗的「批量导入」Tab 新增「📄 从 Word 文档导入（.docx）」按钮：

- **技术实现**：动态加载 JSZip CDN → 以 ArrayBuffer 读取 docx 文件（本质 zip 压缩包）→ 解压 word/document.xml → DOMParser 解析 XML → 按段落合并 `<w:t>` 文本节点 → 正则匹配 `答案：X` 模式提取题目和答案
- **文档格式支持**（来源于 `管理制度.docx` 的实测格式）：
  - 题号行：`1. xxx` → 提取为题目文本（去掉题号和括号出题人）
  - 单选题：选项 A./B./C./D.，答案 `答案：A`
  - 多选题：答案 `答案：ABCD`，入库转为 `A,B,C,D`
  - 判断题：选项 A.对/B.错，答案 `答案：A/B`，入库转为 `true`/`false`
- **容错处理**：去重检测（cleanText 归一化比对）、解析失败提示、跳过无效行
- **预览效果**：导入成功后显示前3条示例及总条数
- **输出文件**：v4.2.0，~1800行
- **参考文档**：`C:/Users/JIA/Downloads/管理制度.docx`（85题，单选/多选/判断三类，答案格式 `答案：X`）

## 踩坑记录 v4.2.1

- **docx导入0条 bug**：根因是 `querySelectorAll('w\\:p')` 在 DOMParser 解析 XML 命名空间文档时失效。
  - **修复**：改用 `doc.getElementsByTagNameNS('*', 'p')` 和 `getElementsByTagNameNS('*', 't')`，用通配符命名空间绕过前缀问题
  - 同时移除无效的 `mergeDocxLines` 合并逻辑，直接在段落级拼接 `<w:t>` 文本
  - `loadJsZipAndParse` 增加 JSZip 复用检测（避免 CDN 加载失败后无 JSZip）和多种 zip 路径容错
  - `showDocxMsg` 改用 `innerHTML` 以支持 HTML 预览格式
  - 新增 `added===0 && skipped===0` 空结果检测
